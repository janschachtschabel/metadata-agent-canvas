## üîÑ Client-Side Routing & Redirects

### Problem: Angular SPA Router

Angular ist eine **Single-Page Application (SPA)**. Wenn ein User eine URL direkt aufruft (z.B. `https://yourapp.netlify.app/events`), muss Netlify die Anfrage zu `index.html` umleiten, damit Angular das Routing √ºbernimmt.

### L√∂sung: SPA Fallback Redirect

**‚úÖ Bereits konfiguriert in `netlify.toml`:**

```toml
[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

**Was das macht:**
- Alle URL-Pfade (`/*`) werden zu `index.html` weitergeleitet
- Status `200` (nicht `301`/`302`) ‚Üí URL bleibt in der Browser-Adresszeile
- Angular Router √ºbernimmt das Client-Side Routing

**Alternative: `_redirects` Datei**

Falls Sie `_redirects` bevorzugen:

```
# src/_redirects
/*    /index.html    200
```

Dann in `angular.json` zur `assets` Array hinzuf√ºgen:

```json
"assets": [
  "src/assets",
  "src/schemata",
  "src/_redirects"  // ‚Üê Neu
]
```

**Hinweis:** Wir nutzen bereits `netlify.toml`, daher ist `_redirects` **nicht n√∂tig**.

## üõ†Ô∏è Troubleshooting

### Problem: 404 bei Page Refresh

**Symptom:** Direct URL wie `/events` gibt 404.

**Ursache:** Netlify kennt die Route nicht (Angular CSR).

**L√∂sung:** Redirects in `netlify.toml` pr√ºfen:
```toml

  from = "/*"
  to = "/index.html"
  status = 200
```

### Problem: Build schl√§gt fehl mit "Budget exceeded"

**Symptom:**
```

```

**L√∂sung:**
```json

```

**Bessere L√∂sung:** Bundle-Gr√∂√üe reduzieren:
- Lazy Loading f√ºr Features
- Tree-shaking pr√ºfen
- Ungenutzte Dependencies entfernen

### Problem: Environment Variables nicht verf√ºgbar

**Ursache:** Netlify Environment Variables nicht gesetzt.

**L√∂sung:**
```bash

```

### Problem: Netlify Functions Timeout

**Symptom:**
```

```

**Ursache:** LLM API Call dauert zu lange.

**L√∂sung:**
```toml

  directory = "netlify/functions"
  node_bundler = "esbuild"

  timeout = 30  # 30 Sekunden (max)
```

**Hinweis:** Free Plan = 10s Timeout, Pro/Business = bis 26s Background Functions.
